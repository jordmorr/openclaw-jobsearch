# Output Format

Use this structure for every shortlist.

## Header

- Employer name
- Portal URL used
- Date checked
- Number of roles reviewed
- Number of roles shortlisted

## Ranked Shortlist

For each shortlisted role, provide:

1. `Rank. Role Title`
2. `Score:` numeric score out of 100
3. `Work model:` remote, hybrid, onsite, or unknown
4. `Location:` verbatim from posting if available
5. `Why it matches:` 2-4 concise evidence-based bullets
6. `Risks or gaps:` 1-2 concise bullets
7. `Source:` direct role URL

## Exclusions Summary

List the main exclusion buckets with counts when practical:
- onsite mismatch
- seniority mismatch
- domain mismatch
- non-technical role
- duplicate or already applied

## Notes

State any limitations such as:
- search results were incomplete
- the portal required heavy client-side rendering
- some listings lacked location or work-model detail
